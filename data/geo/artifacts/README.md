# Geo-артефакты (коммитимые)

Здесь лежит **зафиксированный снапшот** датасетов (файлы + `manifest.json`). Его можно **пушить в git**, чтобы не зависеть от повторных `git clone` при каждом старте.

## Содержимое

- **`manifest.json`** — версия, время сборки, сводка по источникам (git revision), список файлов с SHA-256 и размером.
- **`reference/`** — справочники (например hflabs/region).
- **`boundaries/`** — GeoJSON и сопутствующие файлы из OSM-репозиториев.

## Как обновить снапшот

Из корня репозитория (нужны Git и сеть):

```bash
npm run geo:vendor
npm run geo:sync
```

Затем проверить diff и закоммитить `data/geo/artifacts/`.

## Объём репозитория

Если суммарный размер станет большим, имеет смысл **Git LFS** или вынести тяжёлые слои в отдельный артефакт/CDN — см. обсуждение в команде.

## База данных

Таблица **`geo_dataset_file`** хранет учёт строк по полям манифеста (для связки с FIAS и др. в `meta` позже). После миграций и `geo:sync`:

```bash
npm run migration:run
npm run geo:seed
```
